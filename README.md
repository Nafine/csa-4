## Язык программирования

---

### Форма Бэкуса-Наура

```ebnf
program ::= top_level*

top_level ::= func_decl | var_decl

func_decl ::= "def" ID "(" params? ")" type block

params ::= param ("," param)*
param ::= ID type

block ::= "{" stmt* "}"

stmt ::= var_decl 
    | assign_stmt
    | expr_stmt
    | branch_stmt
    | while_stmt
    | return_stmt

var_decl ::= "var" ID type ("=" expr)? ";"
    | ID ":=" expr ";"

assign_stmt ::= ID "=" expr ";"

branch_stmt ::= "if" "(" expr ")" block 
                ("else" "if" "(" expr ")" block)*
                ("else" block)?

while_stmt ::= "while" "(" expr ")" block

return_stmt ::= "return" expr? ";"
expr_stmt ::= expr ";"

expr ::= expr "||" expr 
    | expr "&&" expr
    | expr ("==" | "!=") expr
    | expr ("<" | ">" | "<=" | ">=") expr 
    | expr ("+" | "-") expr
    | expr ("*" | "/" | "%") expr
    | unary_expr

unary_expr ::= "!" expr 
    | "-" expr
    | postfix_expr

primary_expr ::= ID
    | INT_LIT
    | FLOAT_LIT
    | BOOL_LIT
    | STRING_LIT
    | "(" expr ")"
    | func_call
                 
func_call ::= ID "(" args? ")"
    | "print" "(" expr ")"
    | "read" "(" ")"

args ::= expr ("," expr)*

type ::= "int" | "float" | "bool" | "string" | "void"

INT_LIT ::= [0-9]+
FLOAT_LIT ::= [0-9]+\.[0.9]+
BOOL_LIT ::= "true" | "false"
STRING_LIT ::= \"[^"]*\"
ID ::= [a-zA-Z_][a-zA-Z0-9_]*
```

### Семантика

- Стратегия вычислений -- аппликативная (applicative order)
- Области видимости -- лексические (lexical scoping)
    - Глобальные переменные - объявленные вне тела функций - доступны в любом месте программы
    - Локальные переменные - объявленные внутри функции - доступны только внутри этой функции
    - Затенение (shadowing) - локальная переменная может затенять глобальную переменную с тем же именем
- Типизация -- статическая, явная. Типы: `int`, `float`, `bool`, `string`, `void`
- Литералы -- целые числа (67), числа с плавающей точкой (3.14), булевы значения (true, false), строки ("hello")
- Комментарии -- однострочные (// comment)
- Точки входа -- функция `main` без параметров и возвращаемым значением `void`
- Строки -- null-terminated (C string)
- Ввод-вывод -- через встроенные функции `print` и `read`

## Организация памяти

---

Архитектура - Фон-Неймоновская -- код и данные расположены в едином линейном адресном пространстве.

- Машинное слово - 32 бита
- Адресация -- абсолютная - 24 бита

```
                            Registers
  +-------------------------------------------------------+
  | AC                                                    |
  |-------------------------------------------------------+                          
  | SP                                                    |
  |-------------------------------------------------------+
  | IP                                                    |
  |-------------------------------------------------------+
  | CR                                                    |
  |-------------------------------------------------------+
  | AR                                                    |
  +-------------------------------------------------------+

                            Memory
  +--------------------------------------------------------+
  | 0x000000 | Начало программы                            |
  |   ....   | Инструкции                                  |
  |--------------------------------------------------------+
  |   ....   | статические данные (строки, константы)      |
  |   ....   | глобальные переменные                       |
  |--------------------------------------------------------+
  |          |                                             |
  |   ....   | свободная память                            |
  |          |                                             |
  |--------------------------------------------------------+
  | 0xFFFFFF | Начало стека                                |
  |--------------------------------------------------------+
  
```

### Отображение на память
- Литералы: целочисленные литералы, помещающиеся в 24 бита, помещаются в операнд (LD #67). Литералы большего размера и строковые литералы сохраняются в секцию статических данных.
- Строки (cstr): размещаются в статической памяти посимвольно (один символ — одно 32-битное слово), завершаются нулевым словом.
- Глобальные переменные: размещаются в статической памяти после литералов.
- Локальные переменные: размещаются на стеке при входе в функцию/блок.
- Параметры функций: передаются через стек.
- Инструкции: расположены в памяти начиная с 0x0.

## Система команд

---

### Особенности процессора

- Архитектура: аккумуляторная (acc) -- все операции с данными проходят через регистр ACC.
- Ввод-вывод: 
    - memory-mapped (mem) -- порты ввода/вывода отображены в адресное пространство, взаимодействие напрямую через память (LOAD, STORE).
    - ввод-вывод осуществляется как поток токенов, при старте модели есть буфер, 
в котором представлены все данные ввода (['h', 'e', 'l', 'l', 'o']). Если данные в буфере кончились -- моделирование останавливается.
- Поток управления: инкремент IP после каждой инструкции; условные/безусловные переходы; вызов/возврат из функций через стек.

### Команды

Типы данных в модуле Opcode, Instruction.

| Команда  | Опкод | Такты | Описание                               |
|:---------|:------|-------|:---------------------------------------|
| **LD**   | 0x01  | 2     | AC <- Mem[arg]                         |
| **LDI**  | 0x02  | 1     | AC <- arg                              |
| **ST**   | 0x03  | 2     | Mem[arg] <- AC                         |
| **ADD**  | 0x10  |       | AC <- AC + Mem[arg]                    |
| **SUB**  | 0x11  |       | AC <- AC - Mem[arg]                    |
| **MUL**  | 0x12  |       | AC <- AC * Mem[arg]                    |
| **DIV**  | 0x13  |       | AC <- AC / Mem[arg]                    |
| **REM**  | 0x14  |       | AC <- AC % Mem[arg]                    |
| **CMP**  | 0x20  |       | Set NZ From AC - Mem[arg]              |
| **NOT**  | 0x21  |       | AC <- ~AC                              |
| **NEG**  | 0x22  |       | AC <- ~AC + 1                          |
| **JMP**  | 0x30  |       | IP <- arg                              |
| **BEQ**  | 0x31  |       | IP <- arg if Z = 1 else IP + 1         |
| **BNE**  | 0x32  |       | IP <- arg if Z = 0 else IP + 1         |
| **BGT**  | 0x33  |       | IP <- arg if AC > Mem[arg] else IP + 1 |
| **BLT**  | 0x34  |       | IP <- arg if AC < Mem[arg] else IP + 1 |
| **CALL** | 0x40  |       | PUSH(IP); IP <- arg                    |
| **RET**  | 0x41  |       | IP <- POP()                            |
| **PUSH** | 0x42  |       | MEM[SP] <- AC; SP--                    |
| **POP**  | 0x43  |       | SP++; AC <- Mem[SP]                    |
| **HALT** | 0xFF  |       | 💀                                     |

### Способ кодирования инструкций

Размер инструкции - 32 бита. Формат инструкции:

| Opcode (8 bits) | Operand (24 bits) |
|:----------------|:------------------|

## Транслятор

Использование:
```
python3 cli.py <input.tw> <output.bin>
```

Сначала токенезируется тескт программы, затем строится AST, после чего генерируется машинный код. 
На каждом этапе осуществляется проверка корректности. В конце выдается сгенерированный машинный код в бинарном виде.

Структура бинарного файла - 
- Заголовок - 4 байта - число инструкций (N)
- Инструкции - 4 байта каждая, в формате, описанном выше (Opcode | Argument)
- Секция данных - 3 байта адрес + 4 байта значение

## Модель процессора

- ACC
- MMIO
- Microcode control unit

![Data Path](/docs/CSA%204%20DP.png)
![Control Unit](/docs/CSA%204%20CU.png)

### Data Path

- Защелки
    - acc_l, sp_l, ip_l, cr_l, ar_l -- замкнуть соответствующий регистр
- Управляющие сигналы
    - ar_sel -- выбрать источник для AR (ip, cr)
    - alu_left -- выбрать левый операнд для ALU (acc, sp)
    - alu_right -- выбрать правый операнд для ALU (cr, ip, mem)
    - alu -- выбрать операцию для ALU (ADD, SUB, MUL, DIV, INC, DEC)
    - mem_w -- защелкнуть данные в память по адресу AR
    - cond -- режим условия для переходов

### Control Unit

- Декодер опкода инструкции с маппингом на адрес микрокоманды.
- Память микрокоманд - реализует систему команд.
- Модуль условий для выбора адреса следующей микрокоманды (инкремент или прыжок по next_addr).

### Микрокоманды

1 типа микрокоманды - 24 бита, с битами, отводимыми на сигналы управления и адрес следующей микрокоманды.

| Биты  | Сигнал    | Описание                                              |
|-------|-----------|-------------------------------------------------------|
| 23    | halted    | 1 = остановка симуляции                               |
| 22    | acc_l     | 1 = замкнуть acc                                      |
| 21    | sp_l      | 1 = замкнуть sp                                       |
| 20    | ip_l      | 1 = замкнуть ip                                       |
| 19    | cr_l      | 1 = замкнуть cr                                       |
| 18    | ar_l      | 1 = замкнуть ar                                       |
| 17    | mem_w     | 1 = защелкнуть данные в память по адресу              |
| 16    | ar_sel    | 0 = ip, 1 = cr                                        |
| 15-14 | alu_left  | 00 = 0, 01 = ac, 10 = sp                              |
| 13-12 | alu_right | 00 = 0, 01 = cr, 10 = ip, 11 = mem                    |
| 11-9  | alu       | 000 ADD, 001 SUB, 010 MUL, 011 DIV, 100 INC, 101 DEC  |
| 8-6   | cond      | режим условия                                         |
| 5-0   | next_addr | адрес следующей микрокоманды                          |
